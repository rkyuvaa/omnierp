System Prerequisites

Update System & Install Essentials
# Update package lists and upgrade existing packages
sudo apt update && sudo apt upgrade -y

# Install essential tools
sudo apt install -y curl wget git unzip build-essential software-properties-common
Verify Ubuntu Version
lsb_release -a
# Should show: Ubuntu 24.04 LTS
2

Install PostgreSQL

Install PostgreSQL 16
sudo apt install -y postgresql postgresql-contrib

# Start and enable PostgreSQL to run on boot
sudo systemctl start postgresql
sudo systemctl enable postgresql

# Verify it's running
sudo systemctl status postgresql
# You should see: Active: active (running)
Create Database & User
# Switch to the postgres user
sudo -u postgres psql

# Inside psql — run these 4 commands:
CREATE DATABASE erp_db;
CREATE USER erp_user WITH PASSWORD 'erp_pass';
GRANT ALL PRIVILEGES ON DATABASE erp_db TO erp_user;
ALTER DATABASE erp_db OWNER TO erp_user;

# Exit psql
\q
ℹ
You can change erp_pass to any password you want — just make sure to use the same password in the .env file in Step 4.
Test Database Connection
psql -U erp_user -d erp_db -h localhost -c "SELECT version();"
# Enter password: erp_pass
# Should show PostgreSQL version info
3

Install Python & Backend Dependencies

Install Python 3.11+
# Ubuntu 24.04 ships with Python 3.12 — verify:
python3 --version
# Should show Python 3.12.x or higher

# Install pip and venv
sudo apt install -y python3-pip python3-venv python3-dev libpq-dev
Extract the Project
# Move the downloaded archive to your home folder, then:
cd ~
tar -xzf omni_erp.tar.gz
ls erp/
# Should show: backend/  frontend/  docker-compose.yml  README.md
Create Python Virtual Environment
cd ~/erp/backend

# Create a virtual environment
python3 -m venv venv

# Activate it
source venv/bin/activate
# Your prompt will now show (venv) at the start

# Upgrade pip first
pip install --upgrade pip

# Install all backend dependencies
pip install -r requirements.txt
# This takes 1-3 minutes...
4

Configure & Start the Backend

Create the .env Configuration File
cd ~/erp/backend

# Copy the example config
cp .env.example .env

# Open it for editing
nano .env

Set the contents to:

DATABASE_URL=postgresql://erp_user:erp_pass@localhost:5432/erp_db
SECRET_KEY=my-super-secret-key-change-this-to-something-random-2024
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=480

Press Ctrl+O → Enter → Ctrl+X to save and exit nano.

⚠
Change the SECRET_KEY to any long random string. It's used to sign login tokens. If you leave it as-is, your installation is not secure.
Test-Start the Backend (manual)
cd ~/erp/backend
source venv/bin/activate

# Start the API server (listens on all interfaces)
uvicorn main:app --host 0.0.0.0 --port 8000

# You should see:
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000

# Press Ctrl+C to stop (we will use systemd in Step 8)

Install Node.js & Build the Frontend

Install Node.js 20 (LTS)
# Add NodeSource repository for Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Verify installation
node --version   # Should show v20.x.x
npm --version    # Should show 10.x.x
Install Dependencies & Build
cd ~/erp/frontend

# Install all npm packages
npm install
# Takes 1-2 minutes...

# Build the production bundle
npm run build

# You should see at the end:
✓ built in 5.xx s

# Verify the build output exists
ls dist/
# Should show: index.html  assets/
✓
The frontend is pre-configured to detect the server IP automatically using window.location.hostname. No editing needed even after IP changes.
6

Serve the Frontend (Nginx)

Install Nginx
sudo apt install -y nginx
sudo systemctl start nginx
sudo systemctl enable nginx
Copy Frontend Build to Web Root
# Create the web directory
sudo mkdir -p /var/www/omnierp

# Copy the built files
sudo cp -r ~/erp/frontend/dist/* /var/www/omnierp/

# Set proper ownership
sudo chown -R www-data:www-data /var/www/omnierp
Create Nginx Site Config
sudo nano /etc/nginx/sites-available/omnierp

Paste this entire config:

server {
    listen 80;
    server_name _;   # Accepts any hostname or IP

    root /var/www/omnierp;
    index index.html;

    # Serve React SPA — redirect all routes to index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Proxy API requests to FastAPI backend
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 60s;
    }
}

Press Ctrl+O → Enter → Ctrl+X to save.

Enable the Site
# Enable the site (create symlink)
sudo ln -s /etc/nginx/sites-available/omnierp /etc/nginx/sites-enabled/

# Remove the default site to avoid conflicts
sudo rm -f /etc/nginx/sites-enabled/default

# Test Nginx configuration syntax
sudo nginx -t
# Should show: configuration file test is successful

# Reload Nginx
sudo systemctl reload nginx
ℹ
With this Nginx setup, the frontend is served on port 80 (standard HTTP). Nginx also proxies /api/ requests to the backend, so everything works through one port. The frontend uses window.location.hostname which always resolves to the current IP.

Auto-Start on Reboot (systemd)

This ensures the backend starts automatically every time the server reboots — you won't need to manually start it.

Create systemd Service File
sudo nano /etc/systemd/system/omnierp.service

Paste this content (replace YOUR_USERNAME with your actual Linux username):

[Unit]
Description=OmniERP FastAPI Backend
After=network.target postgresql.service
Requires=postgresql.service

[Service]
Type=simple
User=YOUR_USERNAME
WorkingDirectory=/home/YOUR_USERNAME/erp/backend
Environment=PATH=/home/YOUR_USERNAME/erp/backend/venv/bin
ExecStart=/home/YOUR_USERNAME/erp/backend/venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target

Press Ctrl+O → Enter → Ctrl+X to save.

Find your username if unsure
whoami
# Example output: ubuntu  (or your name like john, dev, etc.)
Enable & Start the Service
# Reload systemd to pick up the new service
sudo systemctl daemon-reload

# Enable it to start on boot
sudo systemctl enable omnierp

# Start it now
sudo systemctl start omnierp

# Check the status
sudo systemctl status omnierp
# Should show: Active: active (running)
Useful Service Commands
# View live logs
sudo journalctl -u omnierp -f

# Restart the backend
sudo systemctl restart omnierp

# Stop the backend
sudo systemctl stop omnierp

Finding Your IP After Every Restart

ℹ
Since your IP changes on every server restart, use any of these methods to find the current IP, then open it in a browser.
Method 1 — ip command (most reliable)
ip addr show | grep "inet " | grep -v "127.0.0.1"

# Example output:
    inet 192.168.1.47/24 brd 192.168.1.255 scope global dynamic ens3
# Your IP is: 192.168.1.47
Method 2 — hostname command
hostname -I | awk '{print $1}'
# Prints just the primary IP: 192.168.1.47
Method 3 — Quick one-liner script (save this)
# Add this alias to ~/.bashrc so you can just type 'myip'
echo "alias myip='hostname -I | awk \"{print \\\$1}\"'" >> ~/.bashrc
source ~/.bashrc

# Now just type:
myip
192.168.1.47
Access the App

Once you have the IP, open a browser on any device on the same network:

# Replace with your actual IP:
http://192.168.1.47/          # ← OmniERP Web App
http://192.168.1.47:8000/docs  # ← API Documentation (Swagger)
✓
Why no IP config is needed: The frontend JavaScript detects window.location.hostname at runtime — the same IP you typed in the browser. It automatically directs all API calls to {that IP}:8000. So even after a restart with a new IP, just open the new IP in your browser and everything works.

Firewall Configuration

Configure UFW Firewall
# Allow SSH (important — do this BEFORE enabling firewall)
sudo ufw allow ssh

# Allow HTTP (port 80) for the web app
sudo ufw allow 80/tcp

# Allow the API port (if you want direct API access)
sudo ufw allow 8000/tcp

# Enable the firewall
sudo ufw enable
# Type 'y' when prompted

# Check status
sudo ufw status

Troubleshooting

Problem: Cannot connect to database
# Test the DB connection manually
psql -U erp_user -d erp_db -h localhost

# If it fails, check PostgreSQL is running
sudo systemctl status postgresql

# Check if pg_hba.conf allows password auth
sudo nano /etc/postgresql/*/main/pg_hba.conf
# Find the line for "local all all" and change "peer" to "md5"
sudo systemctl restart postgresql
Problem: Backend won't start
# View detailed error logs
sudo journalctl -u omnierp -n 50 --no-pager

# Common fix: activate venv and test manually
cd ~/erp/backend && source venv/bin/activate
uvicorn main:app --host 0.0.0.0 --port 8000
# Error message will be visible directly
Problem: Nginx shows 502 Bad Gateway
# Backend is not running — start it
sudo systemctl start omnierp
sudo systemctl status omnierp
Problem: Page loads but login fails (CORS error)
# Check browser console for CORS errors
# The backend allows all origins by default.
# Confirm the backend .env file is correct:
cat ~/erp/backend/.env

# Restart the backend after any .env changes
sudo systemctl restart omnierp
Problem: "Setup already done" error
# The setup endpoint only runs once.
# If you need to reset, connect to the DB and clear tables:
psql -U erp_user -d erp_db -h localhost
TRUNCATE users, roles, branches, modules, stages, sequence_configs RESTART IDENTITY CASCADE;
\q
# Then re-run setup:
curl -X POST http://localhost:8000/api/auth/setup
Problem: Frontend still showing old IP after restart
# The frontend auto-detects IP — just refresh the browser (F5)
# If still not working, hard refresh:
# Chrome/Firefox: Ctrl + Shift + R

# If you rebuilt the frontend, copy dist again:
cd ~/erp/frontend && npm run build
sudo cp -r dist/* /var/www/omnierp/
sudo systemctl reload nginx
Quick Restart Everything After Server Reboot
# All services start automatically on reboot.
# But if you need to manually restart everything:
sudo systemctl restart postgresql
sudo systemctl restart omnierp
sudo systemctl restart nginx

# Then find your new IP:
hostname -I | awk '{print $1}'

# Open in browser:
# http://<NEW_IP>/
🎉 Installation Complete
All services are installed and configured to auto-start on every reboot.
